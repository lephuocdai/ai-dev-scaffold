# Implementation Plans

Implementation plans created during Step 1 of the development lifecycle.

## Naming Convention

Same filename as the corresponding spec: `docs/plans/foo.md` -> `docs/specs/foo.md`

## Template

```markdown
# Plan: {Feature Name}

| Field | Value |
|-------|-------|
| Date | YYYY-MM-DD |
| Roadmap Item | {Phase X, Item Y} |
| Governance | Standard / Security-Critical |
| Status | Draft / Approved / Completed |

## Overview

Brief description of what we are implementing and why.

## Scope

### In Scope
- {What this plan covers}

### Out of Scope
- {What this plan explicitly does NOT cover}

## Affected Modules

| Module | Type of Change |
|--------|---------------|
| {module} | New / Modified / Deleted |

## Acceptance Criteria

- [ ] {Testable criterion 1}
- [ ] {Testable criterion 2}
- [ ] {Testable criterion 3}

## Test Strategy

- Unit tests for: {components}
- Integration tests for: {flows}
- E2E tests for: {user journeys} (if applicable)

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| {risk} | {impact} | {mitigation} |

## Dependencies

- {External dependency or prerequisite}

## Governance Justification

{Why this governance level was chosen. What security/data considerations apply.}
```
